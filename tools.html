Github Audit

Overall the group made good use out of our Github repository; all the changes and comments are reflected well on the final website. Everyone equally has contributed onto the website by pushing their inputs on Github and also adding brief meaningful comments on the changes made. There were moments when there were merge conflicts. To overcome this we decided to pull the latest version of the code before we add modifications and to discuss with other members before pushing the new changes. To improve for next time, we should make use out of other branches for our own modifications and test it, then merge it to the main branch. The benefit of this method is that any changes before it is finalized cannot be seen on the main branch until it is merged.
